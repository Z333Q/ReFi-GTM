# ReFi Trading Go To Market Repository

This repository contains all planning, research, experiments and delivery assets for the ReFi.Trading go‑to‑market process.  It is organised by phase, starting with foundational documents and ending with market launch materials.  Each folder contains Markdown documents or other assets used by the team.

## Structure Overview

The repository is organised into numbered folders reflecting the sequence of work:

- `00-foundation` – Team agreements, initial problem statement, ideal customer profiles and other foundational documents.
- `01-discovery` – Interview guides, user interview notes, synthesis documents and recordings.
- `02-ideation` – Solution concepts, feature prioritisation and MVP definitions.
- `03-prototyping` – Prototype source code and designs, technical decisions, API contracts and data flow diagrams.
- `04-validation` – Usability test plans, test session notes, iteration logs and learnings.
- `05-delivery` – Sales and marketing collateral such as pitch decks, product roadmaps and pricing information.
- `06-go-to-market` – Launch plans, positioning and messaging guides, sales and legal assets and investor data room.
- `07-experiments` – Hypothesis prioritisation, experiment plans, success metrics and other validation work.
- `resources` – General references, meeting notes and tools lists.

Each folder contains placeholder files ready for your content.  Replace the contents with your actual documents as your go‑to‑market work progresses.
